Python Test Driven Development

TDD ou desenvolvimento orientado a testes é uma técnica de desenvolvimento de software que visa a qualidade do software de forma simples objetiva e eficaz. Os conceitos do TDD vieram da mente de Kent Beck e tem conceitos do Extreme Programming.


Ciclos de desenvolvimento


Criar um teste e vê-lo falhar

Escrever um código mínimo porém eficiente para fazer o teste passar

Refatorar o código afim de remover duplicidades,legibilidade e performance

Vantagens de usar TDD


Qualidade do código é sem dúvida um dos principais ensinamentos do TDD, apesar de se ter um curva de aprendizado significativa, com a prática será criada uma cultura entre os programadores, uma vez participado de um projeto orientado a testes, um ciclo será criado e ninguém mais ira programar um função sem ter escrito primeiro o seu teste.

Confiança na equipe é um outro ponto chave do TDD, uma vez que um novo integrante entre na equipe, ou mesmo um desenvolvedor que necessite realizar uma manutenção em um software que não foi ele quem criou, um software desenvolvido orientado a testes trará mais segurança para o sistema e confiança para o desenvolvedor alterar ou refatorar funções sem medo de as mesmas apresetarem bugs posteriormente.


Exemplos


Para exemplificar o uso do TDD será utilizado a biblioteca unittest do python, sendo que inúmeras outras estão presente no mercado.
